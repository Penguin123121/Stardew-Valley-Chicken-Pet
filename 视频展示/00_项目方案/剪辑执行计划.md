# 《星露谷小鸡桌宠》介绍视频执行计划

> **执行要求：** 按任务顺序逐项完成并勾选。每个录制阶段先检查再进入下一阶段，避免集中录完后返工。

**目标：** 完成一条最长不超过 5 分钟、无配音、轻松吐槽风格的 Bilibili 横屏介绍视频，并交付可在剪映专业版 10.9.0 中继续编辑的工程和 1920×1080、60 FPS 成片。

**制作方式：** 主包使用 Windows 11 截图工具按分镜录制 2560×1600 素材；我负责素材验收、剪映粗剪、审核清单、精剪包装和最终导出。游戏标题界面保留现场原声，其他三部分分别使用授权清晰的免版权像素风音乐。

**工具：** Windows 11 截图工具、Xbox Game Bar（仅作为游戏原声录制备选）、剪映专业版 10.9.0、浏览器、Windows 文件资源管理器。

## Task 1：录制环境与隐私准备

**涉及位置：**

- 检查：Windows 桌面、任务栏、通知中心、浏览器、下载文件夹。
- 记录：`视频展示/06_预览与审核/修改意见/录制前检查.md`

- [ ] **Step 1：保持正确显示设置**

  打开 `设置 → 系统 → 屏幕`，确认分辨率为 `2560×1600`、缩放为 `150%`。不要改成 1920×1080 或 100%。

- [ ] **Step 2：关闭可能泄露隐私的程序**

  退出微信、QQ、邮箱、聊天软件、网盘同步提示和 Steam 好友通知。开启 Windows“请勿打扰”，但不要隐藏录制中需要展示的任务栏托盘。

- [ ] **Step 3：整理浏览器**

  使用不显示私人账号的浏览器窗口，关闭书签栏和无关标签页。地址栏必须能完整显示项目仓库域名。

- [ ] **Step 4：整理下载文件夹**

  临时把无关文件移出录制画面，确保打开下载目录时只出现本次下载的 `.exe` 或可公开文件。不要在视频中出现 Windows 用户名路径。

- [ ] **Step 5：检查真实桌面**

  保留日常桌面风格，但确认图标名称、壁纸、任务栏应用和文件夹不包含真实姓名、学校、公司或私人内容。

- [ ] **Step 6：退出已有桌宠实例**

  通过托盘菜单正常退出桌宠，确认任务栏托盘中没有小鸡图标，避免下载演示时触发单实例限制。

- [ ] **Step 7：记录检查结果**

  在 `录制前检查.md` 中记录日期、分辨率、缩放比例、截图工具版本和已完成的隐私检查项。

**验收：** 桌面和浏览器可公开展示；下载目录干净；没有正在运行的桌宠实例。

## Task 2：录制 10 秒测试片

**输出：** `视频展示/01_录制测试/00_录制测试.mp4`

- [ ] **Step 1：打开录屏界面**

  按 `Win + Shift + R`，框选完整屏幕。选择区域时必须覆盖整个 2560×1600 画面，不要只框选单个窗口。

- [ ] **Step 2：录制测试动作**

  开始后静止 2 秒，移动鼠标 2 秒，拖动一个普通窗口 3 秒，再播放一段有系统声音的内容约 3 秒。

- [ ] **Step 3：保存测试片**

  停止录制，将文件保存为 `00_录制测试.mp4`，不要使用截图工具自动生成的默认文件名。

- [ ] **Step 4：检查基础质量**

  播放测试片，确认没有黑屏、画面缺角、鼠标丢失、明显卡顿或通知弹窗。

- [ ] **Step 5：交给我验收**

  我检查实际分辨率、帧率、时长、音轨、编码、16:9 裁切安全区和缩小至 1080p 后的文字清晰度。

- [ ] **Step 6：根据验收结论决定录制工具**

  - 画面和系统声音都正常：所有镜头继续使用截图工具。
  - 画面正常但无系统声音：只有游戏开头改用 Xbox Game Bar。
  - 分辨率或流畅度不合格：暂停正式录制，先调整录制方式并重新生成测试片。

**验收：** 测试片能安全裁成 2560×1440；小鸡动画预计可清楚呈现；游戏原声录制路径已确定。

## Task 3：录制游戏开头

**输出：** `视频展示/02_原始素材/01_游戏开头/01_游戏标题界面_保留原声.mp4`

- [ ] **Step 1：准备游戏**

  使用正版《星露谷物语》，把游戏音量调整到清晰但不过载。关闭会覆盖标题界面的游戏平台通知和帧率显示。

- [ ] **Step 2：开始录制后再进入标题界面**

  先开始录制并静止 3 秒，再让标题界面完整出现。保留 Logo、菜单和背景动画至少 20 秒。

- [ ] **Step 3：不要进行无关操作**

  鼠标停在不遮挡 Logo 和菜单的位置，不进入存档、不打开设置、不显示账号或好友信息。

- [ ] **Step 4：保存并回看原声**

  保存为指定文件名，确认能听到游戏现场音乐，且没有麦克风环境声、通知声或爆音。

- [ ] **Step 5：检查 16:9 裁切**

  确认上下各裁约 80 像素后，Logo、标题菜单和关键动画仍然完整。

**验收：** 至少有连续 15 秒可用标题界面；游戏原声干净；画面可安全裁切。

## Task 4：录制基础行为与鼠标互动

**输出目录：** `视频展示/02_原始素材/02_桌宠功能/`

- [ ] **Step 1：录制行走与发呆**

  保存为 `02_小鸡桌面行走与发呆.mp4`。让小鸡自然行走、转向并至少完整停留一次；总时长建议 45–90 秒，后期挑选片段。

- [ ] **Step 2：录制普通拖拽**

  保存为 `03_普通拖拽.mp4`。静止 3 秒后，缓慢拖动小鸡到画面中央另一位置，松手后再保留 5 秒。

- [ ] **Step 3：录制点击爱心**

  保存为 `04_点击爱心.mp4`。镜头必须包含点击动作、气泡出现、爱心四帧动画、气泡消失和恢复常态。

- [ ] **Step 4：录制长按逃跑**

  保存为 `05_长按逃跑.mp4`。完整保留按住两秒、省略号、挣脱鼠标、逃跑两秒和恢复行为。

- [ ] **Step 5：录制快速拖拽乱麻**

  保存为 `06_快速拖拽乱麻.mp4`。连续快速拖拽约一秒触发表情，表情期间继续拖动，松手后保留恢复画面。

- [ ] **Step 6：检查动作可读性**

  所有镜头必须先让观众看见鼠标动作，再出现桌宠反馈；小鸡不得长期进入上下裁切区。

**验收：** 五类行为分别有一条完整素材；没有把后期音效录入原素材；鼠标动作和反馈关系明确。

- [ ] **Step 2：录制托盘菜单**

- [ ] **Step 3：录制设置参数**

  保存为 `09_设置面板与参数调整.mp4`。依次展示缩放、动画速度、移动速度、活跃度、置顶、穿透和开机自启；只改动当前镜头需要证明的参数。

- [ ] **Step 4：录制活跃度对比**

  保存为 `10_低高活跃度对比.mp4`。分别录制低活跃度和高活跃度，每段至少 30 秒，设置保存后立即开始观察。

- [ ] **Step 5：录制窗口功能**

- [ ] **Step 6：恢复常用设置**

  录制结束后恢复主包平时使用的缩放、速度、活跃度、置顶、穿透和自启设置，避免影响后续镜头。

**验收：** 喂食动画完整；设置保存立即生效的证据清楚；托盘区域可通过单独裁切完整显示。

## Task 8：统一素材验收与补录

**创建：** `视频展示/06_预览与审核/修改意见/素材验收报告.md`

- [ ] **Step 1：检查文件齐全性**

  对照 `录制素材清单.md` 检查 `00` 至 `17`，记录缺失、重复或命名错误。

- [ ] **Step 2：检查技术参数**

  逐条检查分辨率、帧率、音轨、时长、编码和文件是否可正常解码。

- [ ] **Step 3：检查隐私和安全**

  检查每条素材是否包含账号、通知、真实姓名、Windows 用户名、私人文件、下载历史或聊天内容。

- [ ] **Step 4：检查内容完整性**

  标记动作被截断、鼠标操作不清楚、小鸡被遮挡、文字模糊、原声缺失和裁切不安全的镜头。

- [ ] **Step 5：生成精确补录清单**

  每项写明文件名、问题发生时间、需要重录的动作、建议时长和正确示例。主包用 `_02` 后缀补录，不覆盖旧文件。

- [ ] **Step 6：通过验收门槛**

  只有核心镜头全部合格后才进入粗剪；不使用模糊、泄露隐私或动作不完整的素材强行拼接。

**验收：** `素材验收报告.md` 无未解决的“阻塞”项。

## Task 9：选择音乐和后期音效

**涉及位置：**

- 保存：`视频展示/03_音乐音效/02_免版权音乐/`
- 保存：`视频展示/03_音乐音效/03_后期音效/`
- 更新：`视频展示/03_音乐音效/素材授权与来源.md`

- [ ] **Step 1：选择第一部分音乐**

  选择活泼、俏皮、节奏较强的像素风音乐，必须允许 Bilibili 视频使用和平台收益场景。

- [ ] **Step 2：选择第二部分音乐**

  选择轻快、规整、存在感较低的像素风音乐，确保不会干扰 GitHub 操作字幕。

- [ ] **Step 3：选择第三部分音乐**

  选择温暖、有收束感的像素风音乐，结尾应能自然淡出。

- [ ] **Step 4：选择低频剪辑音效**

  只准备章节提示、文字弹出、局部放大和夸张转场所需音效；不准备脚步、咀嚼、鼠标点击和滑块等拟真声音。

- [ ] **Step 5：记录许可**

  每项素材填写名称、作者、直接来源链接、许可类型、署名要求、下载日期和商用/平台收益适用情况。许可不明确的素材不得进入工程。

**验收：** 三首音乐和全部音效均有可追溯许可记录；视频不会让观众误以为桌宠自带音效。

## Task 10：建立剪映 10.9.0 粗剪工程

**输出：** `视频展示/05_剪映工程/粗剪/星露谷小鸡桌宠_粗剪`

- [ ] **Step 1：新建项目**

  设置画布为 16:9，项目目标为 1920×1080、60 FPS。不要直接使用 16:10 画布。

- [ ] **Step 2：导入原始素材**

  按 `01` 至 `17` 顺序导入，不改变原始文件位置，不使用剪映代理文件替代交付源文件。

- [ ] **Step 3：统一裁切**

  普通镜头从 2560×1600 居中裁为 2560×1440；托盘和任务栏镜头单独向下调整；逐镜头检查关键内容。

- [ ] **Step 4：搭建四段时间线**

  按“独立开头 → 第一部分 → 第二部分 → 第三部分”建立主时间线，目标总时长 3 分 30 秒至 4 分 30 秒，绝不超过 5 分钟。

- [ ] **Step 5：加入临时字幕**

  使用设计方案中的已确认字幕；每屏不超过两行，关键操作至少停留 1.5 秒。先使用统一基础字体，不在粗剪阶段堆叠复杂动画。

- [ ] **Step 6：加入临时音乐**

  游戏开头保留现场原声；三部分使用已选免版权音乐。音乐切换处只做基础淡入淡出。

- [ ] **Step 7：加入音效说明**

  第一部分开头显示约 3 秒：“本视频音效为后期剪辑添加，桌宠当前暂无内置音效功能。音响系统：还在鸡蛋里 🥚”。

- [ ] **Step 8：保存工程副本**

  保留一个未精剪的粗剪版本，后续任何修改都基于复制出的精剪工程，避免破坏审核基线。

**验收：** 工程可在剪映专业版 10.9.0 正常打开；四段顺序正确；无离线素材；总时长不超过 5 分钟。

  主包按“时间点 + 当前问题 + 希望修改成什么”的格式反馈。涉及叙事结构的修改在此阶段完成，不拖到精剪后返工。

- [ ] **Step 5：锁定粗剪**

  只有主包明确回复“粗剪确认”后，复制工程进入精剪目录。

**验收：** 主包确认内容顺序、镜头取舍、字幕意思、音乐方向和整体节奏。

## Task 12：精剪与视觉包装
